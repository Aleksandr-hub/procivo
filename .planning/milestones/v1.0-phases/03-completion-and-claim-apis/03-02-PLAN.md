<objective>
Add pessimistic locking to claim/unclaim handlers to prevent the double-claim race condition, and add comprehensive unit tests.

Purpose: Pool tasks can be claimed by multiple eligible employees simultaneously. Without database-level locking, concurrent claims create a TOCTOU race condition where two users both see assigneeId=null and both claim the task.

Output: `findByIdForUpdate()` with `PESSIMISTIC_WRITE` lock in repository, transaction-wrapped claim/unclaim handlers, and unit tests for both handlers.
</objective>

**Step 3: Upgrade ClaimTaskHandler with transaction + lock**

1. Add `EntityManagerInterface` to constructor:
   ```php
   public function __construct(
       private TaskRepositoryInterface $taskRepository,
       private OrganizationQueryPort $organizationQueryPort,
       private EntityManagerInterface $entityManager,
   ) {}
   ```

2. Wrap the entire `__invoke` body in `$this->entityManager->wrapInTransaction()`:
   ```php
   public function __invoke(ClaimTaskCommand $command): void
   {
       $this->entityManager->wrapInTransaction(function () use ($command): void {
           $task = $this->taskRepository->findByIdForUpdate(
               TaskId::fromString($command->taskId),
           );

           if (null === $task) {
               throw TaskNotFoundException::withId($command->taskId);
           }

           if (!$task->isPoolTask()) {
               throw TaskClaimException::notAPoolTask($command->taskId);
           }

           if (null !== $task->assigneeId()) {
               throw TaskClaimException::alreadyClaimed($command->taskId);
           }

           $this->validateEligibility($command->employeeId, $task);

           $task->claim($command->employeeId);
           // No explicit save — wrapInTransaction calls flush() before commit
       });
   }
   ```

3. Add import: `use Doctrine\ORM\EntityManagerInterface;`

4. Keep the `validateEligibility()` private method unchanged.

IMPORTANT: Do NOT call `$this->taskRepository->save($task)` inside wrapInTransaction. The `wrapInTransaction()` method calls `flush()` before commit automatically. Double-flushing can cause unexpected behavior.

   ```php
               TaskId::fromString($command->taskId),

           if (!$task->isPoolTask()) {

           }

           if ($task->assigneeId() !== $command->employeeId) {
               throw TaskClaimException::notClaimed($command->taskId);
           }

           $task->unclaim();
           // No explicit save — wrapInTransaction calls flush() before commit
       });
   }
   ```
