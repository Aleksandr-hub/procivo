       **Job 2: `frontend`**
       - `runs-on: ubuntu-latest`
       - Steps:
         - `actions/checkout@v4`
         - `actions/setup-node@v4` with node-version: '22', cache: 'npm', cache-dependency-path: `frontend/package-lock.json`
         - `Install dependencies`: `working-directory: frontend`, `run: npm ci`
         - `Type check`: `working-directory: frontend`, `run: npm run type-check`
         - `ESLint`: `working-directory: frontend`, `run: npm run lint:eslint`

       NOTE: Use `lint:eslint` (not `lint`) in CI to avoid `oxlint --fix` which auto-fixes in place â€” we want CI to detect, not fix. ESLint `--fix` is also fine in CI because it exits non-zero when there are unfixable issues, but if we want pure detection use `eslint . --cache` without `--fix`. Decision: use `npx eslint .` (without --fix) for CI. Separate step from the package.json lint script which uses --fix.

    2. **Create `lefthook.yml`** at project root:
       ```yaml
       pre-commit:
         parallel: false
         commands:
           php-cs-fixer:
             glob: "backend/**/*.php"
             run: docker compose exec -T php vendor/bin/php-cs-fixer fix {staged_files} --config=.php-cs-fixer.dist.php
             stage_fixed: true

           eslint:
             glob: "frontend/**/*.{ts,vue}"
             run: cd frontend && npx eslint --fix {staged_files}
             stage_fixed: true
       ```

       Notes:
       - `parallel: false` ensures sequential execution (CS Fixer, then ESLint).
       - `stage_fixed: true` re-stages auto-fixed files so the commit includes the fixes.
       - `-T` flag on docker exec prevents TTY allocation errors in non-interactive contexts.
       - The php-cs-fixer `--config` path is relative to the container's WORKDIR (/var/www), which maps to `./backend/`.
       - ESLint runs natively (not via Docker) since Node.js is available on the host.

    3. **Install lefthook** â€” lefthook needs to be available. Since there's no root `package.json`, the README will instruct developers to install lefthook globally (`brew install lefthook` or `npm install -g lefthook`) and run `lefthook install` to set up git hooks.
