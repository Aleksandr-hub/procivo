Create the backend stats endpoint following the established DBAL query handler pattern (like ListAuditLogHandler).

**GetDashboardStatsQuery.php:**
- Readonly class with `public readonly string $organizationId` constructor parameter
- Namespace: `App\TaskManager\Application\Query\GetDashboardStats`

**GetDashboardStatsHandler.php:**
- `#[AsMessageHandler(bus: 'query.bus')]`
- Constructor: `private Connection $connection`
- `__invoke(GetDashboardStatsQuery $query): array` — returns associative array with 3 aggregates:

1. **tasks_by_status**: `SELECT status, COUNT(*) as cnt FROM task_manager_tasks WHERE organization_id = :orgId GROUP BY status` → returns `array<string, int>` via `fetchAllKeyValue()`

2. **tasks_completed_by_day**: `SELECT DATE(updated_at) as day, COUNT(*) as cnt FROM task_manager_tasks WHERE organization_id = :orgId AND status = 'done' AND updated_at >= :since GROUP BY DATE(updated_at) ORDER BY day` with `:since` = `-30 days` in `Y-m-d` format → returns `list<array{day: string, cnt: int}>` via `fetchAllAssociative()`. Cast cnt to (int) for each row.

3. **processes_by_status**: `SELECT status, COUNT(*) as cnt FROM workflow_process_instances_view WHERE organization_id = :orgId GROUP BY status` → returns `array<string, int>` via `fetchAllKeyValue()`

Return format: `['tasks_by_status' => [...], 'tasks_completed_by_day' => [...], 'processes_by_status' => [...]]`

**DashboardController.php:**
- Namespace: `App\TaskManager\Presentation\Controller`
- `#[Route('/api/v1/organizations/{organizationId}/dashboard', name: 'api_v1_dashboard_')]`
- `final readonly class DashboardController` with `QueryBusInterface` and `OrganizationAuthorizer` constructor injection
- Single method `stats(string $organizationId): JsonResponse`:
  - `$this->authorizer->authorize($organizationId, 'TASK_VIEW')` (reuse TASK_VIEW permission — dashboard shows task and process data)
  - `$stats = $this->queryBus->ask(new GetDashboardStatsQuery($organizationId))`
  - `return new JsonResponse($stats)`

Use imports: `Symfony\Component\HttpFoundation\JsonResponse`, `Symfony\Component\Routing\Attribute\Route`, `App\Shared\Application\Bus\QueryBusInterface`, `App\Shared\Infrastructure\Security\OrganizationAuthorizer`.

**dashboard.store.ts:**
Create a Pinia store `defineStore('dashboard', () => { ... })` with:

State refs:
- `myTasks: ref<TaskDTO[]>([])` — imported from `@/modules/tasks/types/task.types`
- `activeProcesses: ref<ProcessInstanceDTO[]>([])` — imported from `@/modules/workflow/types/process-instance.types`
- `stats: ref<DashboardStatsDTO | null>(null)`
- `recentActivity: ref<AuditLogDTO[]>([])` — imported from `@/modules/audit/types/audit-log.types`
- `loading: ref(false)`

Actions:
- `async fetchAll(orgId: string, employeeId: string | null)` — sets `loading = true`, then `await Promise.all([...])` calling 4 private fetch functions, then `loading = false` in finally block.

Private fetch functions (called inside Promise.all):
1. `fetchMyTasks(orgId, employeeId)` — if `employeeId` is not null, call `taskApi.list(orgId, undefined, employeeId)` and filter result to exclude `done` and `cancelled` status. Store in `myTasks.value`. If `employeeId` is null, set `myTasks.value = []`.
2. `fetchActiveProcesses(orgId)` — call `processInstanceApi.list(orgId, { status: 'running' })`, extract `.items`, store in `activeProcesses.value`. NOTE: No frontend filter by started_by here — the widget will show all running processes for the org (user participation scope is applied at the widget level if needed later).
3. `fetchStats(orgId)` — call `dashboardApi.stats(orgId)`, store in `stats.value`.
4. `fetchRecentActivity(orgId)` — call `auditLogApi.list(orgId, { limit: 20 })`, extract `.items`, store in `recentActivity.value`.

Each individual fetch should have its own try/catch so one failure doesn't block others.

Return all refs and `fetchAll` from the store.
