**DashboardPage.vue:**
- Script setup with `useRoute()`, `useAuthStore()`, `useEmployeeStore()`, `useDashboardStore()`
- Extract `orgId` from `route.params.orgId` as string
- Compute `currentEmployeeId`: `empStore.employees.find(e => e.userId === authStore.user?.id && e.status === 'active')?.id ?? null` — this resolves user ID to employee ID (CRITICAL: task API uses employee IDs, not user IDs)
- `onMounted`: call `dashboardStore.fetchAll(orgId, currentEmployeeId.value)`
- Template: 2x2 CSS grid (locked decision) with 4 PrimeVue `<Card>` components, each containing one widget
- Grid CSS: `.dashboard-grid { display: grid; grid-template-columns: repeat(2, 1fr); gap: 1.5rem; padding: 1.5rem; }` with `@media (max-width: 768px) { grid-template-columns: 1fr; }`
- Each card has a header slot with `<template #title>{{ t('dashboard.xxx') }}</template>`
- Show `<Skeleton>` placeholders (height 200px) while `dashboardStore.loading` is true
- Pass data from store to each widget via props

**MyTasksWidget.vue (DASH-01):**
- Props: `tasks: TaskDTO[]`, `orgId: string`
- Computed buckets using `isOverdue()` from `@/shared/utils/date-format`:
  - `overdueTasks`: tasks where `dueDate` is set and `isOverdue(dueDate)` is true
  - `todayTasks`: tasks where `dueDate` is set and date part equals today (compare `new Date(dueDate).toDateString() === new Date().toDateString()`)
  - `upcomingTasks`: tasks where `dueDate` is set, is in the future, and not today
  - `noDueDateTasks`: tasks where `dueDate` is null
- Each bucket has a section header (e.g., "Overdue (3)") colored appropriately: red for overdue, orange for today, blue for upcoming, gray for no due date
- Each task renders as a clickable row/card: title, priority Tag (severity: critical=danger, high=warn, medium=info, low=secondary), dueDate formatted, and an optional workflow badge if `workflow_summary` exists
- Click navigates to `router.push({ name: 'task-detail', params: { orgId, taskId: task.id } })`
- Empty state: centered icon `pi pi-check-circle` with text "No tasks assigned to you"
- Use PrimeVue `<Tag>` for priority badges

**ActiveProcessesWidget.vue (DASH-02):**
- Props: `processes: ProcessInstanceDTO[]`, `orgId: string`
- Render a list of process cards, each showing:
  - `definition_name` as title
  - Status `<Tag>`: running=info, completed=success, cancelled=secondary, error=danger
  - `started_at` formatted via `formatRelativeDate()`
  - Current token node info if tokens array is non-empty (optional: show first active token status)
- Click navigates to `router.push({ name: 'process-instance-detail', params: { orgId, instanceId: process.id } })`
- Limit display to first 10 items. If more, show "and N more..." link
- Empty state: `pi pi-play` icon with text "No active processes"

**ChartsWidget.vue (DASH-03):**
- Props: `stats: DashboardStatsDTO | null`
- Render 3 charts inside a flex layout (column direction with gap):

1. **Donut chart (tasks by status):**
   - Labels: map `stats.tasks_by_status` keys to localized status labels using `t()`. Use existing i18n keys if available or add new ones.
   - Data: `Object.values(stats.tasks_by_status)`
   - Colors: open=blue-400, in_progress=orange-400, review=purple-400, done=green-400, blocked=red-400, draft=surface-400, cancelled=surface-300
   - Use PrimeVue CSS variable values: `getComputedStyle(document.documentElement).getPropertyValue('--p-blue-400')` resolved at mount time and stored in a ref for chart colors. This ensures dark mode compatibility.
   - Chart options: `{ responsive: true, maintainAspectRatio: false, plugins: { legend: { position: 'bottom', labels: { color: textColor } } } }` where `textColor` is resolved from `--p-text-color`

2. **Line chart (completed over time - last 30 days):**
   - Labels: `stats.tasks_completed_by_day.map(d => d.day)` — format as short date
   - Data: `stats.tasks_completed_by_day.map(d => d.cnt)`
   - Line color: `--p-primary-color`, tension 0.4, fill false
   - Chart options with scales: x/y ticks color from `--p-text-muted-color`

3. **Bar chart (process completion rate):**
   - Labels: Running, Completed, Cancelled (localized)
   - Data: from `stats.processes_by_status` with keys 'running', 'completed', 'cancelled'
   - Colors: running=blue-400, completed=green-400, cancelled=surface-400
   - Chart options with scales

- If `stats` is null, show "No data yet" empty state
- Each chart inside a div with `max-height: 220px` to keep visual balance
- Use `<Chart type="doughnut" :data="..." :options="..." />`, `<Chart type="line" .../>`, `<Chart type="bar" .../>`

**RecentActivityWidget.vue (DASH-04):**
