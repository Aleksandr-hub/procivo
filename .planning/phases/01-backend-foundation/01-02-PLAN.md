```php
{
    public function __construct(
        public string $field,
        public string $rule,       // 'required', 'type', 'min', 'max', 'minLength', 'maxLength', 'pattern'

}
```

```php
final class FormSchemaValidator
{
    /**
     * @param list<array<string, mixed>> $fields   Form field definitions
     * @param array<string, mixed>       $formData Submitted data
     * @return list<FieldValidationError>          Empty = valid
     */
    public function validate(array $fields, array $formData): array
```

Implementation structure:
1. `resolveFieldDependencies(array $fields, array $formData): array` — Iterate through fields. For each field with `dependsOn`, check if the dependency condition is met. If not met, mark as not-effective (skip validation). Use iterative resolution with max 10 iterations for cascading deps. Return only effective (visible) fields.
2. Main `validate` loop: for each effective field, check required → type → constraints in order. Stop after first error per field for `required` (continue for other rules).
3. `validateType(string $type, mixed $value): ?FieldValidationError` — Switch on type: text/textarea (is_string), number (is_numeric), date (strtotime or DateTime parse), select (in_array against options), checkbox (boolean-ish check), employee (is_string — just an ID).
4. `validateConstraints(array $field, mixed $value): array` — Check min, max (numeric), minLength, maxLength (string strlen), pattern (preg_match).
