<objective>
Add event-sourced migration semantics to ProcessInstance, enhance compatibility validation with action_key checks, and extend the versions API with running instance counts.

Purpose: The current migration handler only updates the read model — after migration, the engine still reconstitutes instances from the event store with the original versionId. This plan introduces a `ProcessInstanceMigratedEvent` so migration actually changes which version the engine uses for execution. Additionally, the action_key compatibility check prevents migrating instances to versions where active task nodes lost their outgoing actions.

Output: Working backend migration that updates both event store and read model, enhanced validation, and running instance count in version listing API.
</objective>

         ```php

         foreach ($runningRows as $row) {
             $tokens = json_decode((string) $row['tokens'], true, 512, JSON_THROW_ON_ERROR);
             foreach ($tokens as $token) {
                 if (isset($token['node_id'], $token['status']) && 'completed' !== $token['status']) {
                     // Existing check: node exists
                     if (!in_array($token['node_id'], $targetNodeIds, true)) {
                         throw new \DomainException(sprintf('Cannot migrate instance %s: active token at node "%s" which does not exist in target version %d.', $row['id'], $token['node_id'], $targetVersion->versionNumber()));
                     }
                     // New check: for task nodes, verify outgoing transitions have action_keys
                     if ('task' === $targetGraph->nodeType($token['node_id'])) {
                         $outgoing = $targetGraph->outgoingTransitions($token['node_id']);
                         if (0 === count($outgoing)) {
                             throw new \DomainException(sprintf('Cannot migrate instance %s: task node "%s" has no outgoing transitions in target version %d.', $row['id'], $token['node_id'], $targetVersion->versionNumber()));
                         }
                     }
                 }
             }
         }
         ```
