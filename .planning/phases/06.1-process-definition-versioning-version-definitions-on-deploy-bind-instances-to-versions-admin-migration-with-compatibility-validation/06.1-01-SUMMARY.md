## Decisions Made

- Migration is event-sourced: ProcessInstanceMigratedEvent stored in event store, projection handles read model update — command handler has no direct SQL UPDATE
- Action_key validation checks outgoing transitions count > 0, not exact key matching — avoids expensive per-instance current version load during validation loop
- DTO uses default value `int $runningInstanceCount = 0` — backward compatible, existing callers unaffected
- Used DBAL `IN(:ids)` with `ArrayParameterType::STRING` (not PostgreSQL `ANY()`) — DBAL expands array binding automatically and is DB-agnostic
