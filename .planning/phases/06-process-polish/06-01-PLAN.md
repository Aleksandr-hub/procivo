<objective>
Refactor ProcessDefinition publish flow to allow re-publishing without draft revert, dedup FormSchemaBuilder, serve form_schema from snapshot, and add FromVariable enum.

Purpose: The current flow forces revert-to-draft before editing a published definition. This is wrong for proper versioning — a published definition should stay published while users edit and re-deploy. Each publish creates a new immutable ProcessDefinitionVersion. Also fixes three backend tech debts: duplicated form schema logic, live-read instead of snapshot, and missing FromVariable case.

Output: ProcessDefinition allows re-publish from Published state. Backend handlers use FormSchemaBuilder exclusively. Task detail serves snapshot. AssignmentStrategy handles all cases.
</objective>

**ProcessDefinition.php — Allow publish from Published state:**

Change the `publish()` method to allow publishing from both Draft and Published states. A published definition should be re-publishable (each publish creates a new version). Only Archived definitions cannot be published.

Replace:
```php
public function publish(string $versionId, int $versionNumber, string $publishedBy): void
{
    if (!$this->isDraft()) {
        throw new \DomainException('Only draft definitions can be published.');
    }
    // ...
}
```

With:
```php
public function publish(string $versionId, int $versionNumber, string $publishedBy): void
{
    if (ProcessDefinitionStatus::Archived === $this->status()) {
        throw new \DomainException('Archived definitions cannot be published.');
    }

    $this->status = ProcessDefinitionStatus::Published->value;
    $this->updatedAt = new \DateTimeImmutable();

    $this->recordEvent(new ProcessDefinitionPublishedEvent(
        $this->id,
        $versionId,
        $versionNumber,
        $publishedBy,
    ));
}
```

This means: Draft → Published (first publish) and Published → Published (re-publish with new version) are both valid. Running instances stay on their version. The "revert to draft" method can remain for backward compatibility but is no longer required for the edit-re-publish flow.

**PublishProcessDefinitionHandler.php — no changes needed** — it already creates a new version with `$latestVersion->versionNumber() + 1`. The only change was in the entity's `publish()` guard, done above.
