**ListVersionsHandler.php — create handler:**

```php
#[AsMessageHandler(bus: 'query.bus')]
final readonly class ListVersionsHandler
{
    public function __construct(
        private ProcessDefinitionVersionRepositoryInterface $versionRepository,
    ) {}

    /**
     * @return list<ProcessDefinitionVersionDTO>
     */
    public function __invoke(ListVersionsQuery $query): array
    {
        $defId = ProcessDefinitionId::fromString($query->processDefinitionId);
        $versions = $this->versionRepository->findByProcessDefinitionId($defId);

        // Sort by version number descending (newest first)
        usort($versions, static fn ($a, $b) => $b->versionNumber() <=> $a->versionNumber());

        return array_map(
            static fn (ProcessDefinitionVersion $v) => ProcessDefinitionVersionDTO::fromEntity($v),
            $versions,
        );
    }
}
```

**MigrateProcessInstancesHandler.php — create handler:**

This handler migrates ALL running process instances of a given definition to a target version. Migration updates the `version_id` column in the event store or read model. For now, implement a basic migration that:

1. Loads the target version to verify it exists
2. Finds all running instances of this definition via DBAL query against the read model (`workflow_process_instances_view` or `workflow_process_instance_events` table)
3. Updates the `version_id` for running instances

Important: For the v2.0 scope, implement a simple version swap — no schema compatibility validation yet. The handler should:
```php
#[AsMessageHandler(bus: 'command.bus')]
final readonly class MigrateProcessInstancesHandler
{
    public function __construct(
        private ProcessDefinitionVersionRepositoryInterface $versionRepository,
        private Connection $connection,
    ) {}

    public function __invoke(MigrateProcessInstancesCommand $command): void
    {
        $targetVersion = $this->versionRepository->findById(
            ProcessDefinitionVersionId::fromString($command->targetVersionId),
        );

        if (null === $targetVersion) {
            throw new \DomainException('Target version not found.');
        }

        // Update version_id for all running instances of this definition
        // This updates the read model view — the event store retains the original version in ProcessStartedEvent
        $this->connection->executeStatement(
            'UPDATE workflow_process_instance_events
             SET payload = jsonb_set(payload, \'{versionId}\', to_jsonb(:targetVersionId::text))
             WHERE event_type = \'process.started\'
             AND payload->>\'processDefinitionId\' = :definitionId
             AND (SELECT status FROM workflow_process_instances_view WHERE id = workflow_process_instance_events.process_instance_id) = \'running\'',
            [
                'targetVersionId' => $command->targetVersionId,
                'definitionId' => $command->processDefinitionId,
            ],
        );
    }
}
```

NOTE: The exact SQL depends on the event store schema. Check the actual table structure by examining `DoctrineProcessInstanceEventStore.php` or the migration files. If the event store uses a different column structure, adapt accordingly. The key principle: update the versionId in the ProcessStartedEvent payload for running instances so that when the instance is reconstituted, it reads from the new version.
