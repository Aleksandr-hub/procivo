## Decisions Made

- **resolveEmployeeDisplayNames breaking change scoped**: Only ListTasksHandler and GetTaskHandler call this method — both updated. No other callers exist.
- **AvatarStorageInterface in TaskManager infrastructure**: The adapter already injected UserRepositoryInterface from Identity module — adding AvatarStorageInterface follows same cross-module pattern within infrastructure layer only.
- **Batch vs per-row for comment counts**: Used DBAL `fetchAllKeyValue` with `IN(?)` for list handler (N tasks), `fetchOne` for single task handler — appropriate sizing.
- **KanbanCard dragstart emit**: Parent tracks `draggedTaskId` state, so `dragstart` is emitted up rather than handled internally.
