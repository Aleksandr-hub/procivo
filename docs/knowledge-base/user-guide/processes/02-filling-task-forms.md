## Огляд

У бізнес-процесах Procivo кожна дія (transition) над завданням може мати власну форму. Форми дозволяють збирати дані від виконавця перед виконанням дії — наприклад, вказати причину відхилення або ввести суму оплати. Дані форми зберігаються в контексті процесу.

## Кроки

### Відкриття форми дії

1. Перейдіть до процесного завдання (через список завдань, дашборд або сповіщення).
2. На сторінці завдання ви побачите доступні **дії** — кнопки з назвами переходів (наприклад, «Затвердити», «Відхилити», «Повернути на доопрацювання»).
3. Натисніть на потрібну дію.
4. Якщо дія має форму, з'явиться діалог **ActionFormDialog** з полями для заповнення.

### Типи полів форми

| Тип поля | Опис | Приклад |
|----------|------|---------|
| **Текст** (text) | Однорядкове текстове поле | Ім'я, коментар |
| **Число** (number) | Числове поле з валідацією | Сума, кількість |
| **Дата** (date) | Вибір дати з календаря | Дата виконання |
| **Вибір** (select) | Випадаючий список з варіантами | Категорія, статус |
| **Чекбокс** (checkbox) | Прапорець так/ні | Погодження, підтвердження |

### Заповнення та валідація

1. Заповніть усі обов'язкові поля (позначені зірочкою).
2. Система перевіряє дані в реальному часі:
   - Обов'язкові поля не можуть бути порожніми.
   - Числа перевіряються на допустимий діапазон.
   - Текст може мати обмеження за довжиною або форматом (regex).
   - Деякі поля мають залежності — значення одного може впливати на доступність іншого.
3. Якщо валідація не пройдена, під полем з'являється повідомлення про помилку.

### Відправка форми

1. Після заповнення натисніть **«Виконати»** (або назву дії).
2. Дані форми зберігаються в контексті процесу.
3. Процес переходить до наступного етапу згідно з BPMN-схемою.
4. Якщо процес має XOR-розгалуження, маршрут визначається автоматично на основі введених даних.

## Поради

- Різні дії мають різні форми — наприклад, «Затвердити» може не мати форми, а «Відхилити» може вимагати причину.
- Дані форми доступні в контексті процесу для подальших етапів та умов маршрутизації.
- Якщо ви помилилися, зверніться до менеджера — деякі процеси дозволяють повернення на попередній етап.
- Форми підтримують залежності між полями: наприклад, вибір «Тип документа» може змінити набір додаткових полів.
