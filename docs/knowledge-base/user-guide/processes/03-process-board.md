## Огляд

Процесна дошка — це спеціальний тип дошки, де колонки відповідають етапам (вузлам) BPMN-процесу, а картки — це екземпляри процесу. Вона створюється на основі опублікованого визначення процесу і дозволяє бачити, на якому етапі знаходиться кожен екземпляр.

## Кроки

### Створення процесної дошки

1. Перейдіть до розділу **«Дошки»** у бічній панелі.
2. Натисніть **«Створити дошку»**.
3. Оберіть тип **«Процесна дошка»** (Process Board).
4. Оберіть **опубліковане визначення процесу** зі списку.
5. Система автоматично створить колонки на основі етапів BPMN-схеми.
6. Натисніть **«Створити»**.

### Структура дошки

1. **Колонки** розташовані зліва направо у **топологічному порядку** етапів BPMN-процесу.
2. Кожна **картка** представляє один екземпляр процесу.
3. На картці відображається:
   - Ідентифікатор або назва екземпляра.
   - Поточний статус.
   - Призначений виконавець поточного завдання.
4. Заголовок кожної колонки показує кількість активних екземплярів на цьому етапі.

### Drag-to-Complete

1. Перетягніть картку в наступну колонку для виконання дії.
2. Якщо перехід має форму, з'явиться **ActionFormDialog** — заповніть та підтвердіть.
3. Якщо перехід без форми, екземпляр автоматично переходить на наступний етап.
4. Переміщення дозволено тільки між сусідніми етапами згідно з BPMN-схемою.

### Метрики пайплайну

1. У верхній частині дошки відображаються метрики:
   - **Загальна кількість** активних екземплярів.
   - **Sparkline** — мініатюрний графік завершених процесів за останній час.
2. Метрики оновлюються автоматично при переміщенні карток.

## Поради

- Процесна дошка — це найзручніший спосіб бачити стан усіх екземплярів одного процесу.
- Drag-to-complete працює через Workflow Engine — усі бізнес-правила та валідації зберігаються.
- Одне визначення процесу може мати кілька дошок з різними фільтрами.
- Використовуйте процесну дошку разом зі стрічкою активності для повної картини.
