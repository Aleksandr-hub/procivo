## Огляд

Контроль доступу до процесів (ACL) дозволяє обмежити, які відділи та ролі можуть запускати або переглядати конкретне визначення процесу. За замовчуванням процеси доступні всім -- обмеження застосовуються лише при явному налаштуванні.

## Модель доступу (Whitelist)

Система працює за принципом білого списку:

- **Немає записів** -- процес доступний всім працівникам організації
- **Є записи** -- процес доступний лише вказаним відділам та ролям
- **Власник** процесу завжди має повний доступ, незалежно від ACL

## Налаштування доступу

1. Відкрийте список визначень процесів
2. Знайдіть потрібний процес
3. Натисніть іконку налаштування доступу (замок)
4. Відкриється діалог ProcessAccessDialog

### Додавання обмежень

1. У діалозі оберіть тип обмеження:
   - **Відділ** -- всі працівники обраного відділу отримують доступ
   - **Роль** -- всі працівники з обраною роллю отримують доступ
2. Оберіть конкретний відділ або роль з випадаючого списку
3. Натисніть "Додати"
4. Повторіть для додаткових обмежень
5. Натисніть "Зберегти"

### Видалення обмежень

1. Відкрийте діалог доступу до процесу
2. Знайдіть запис, який потрібно видалити
3. Натисніть іконку видалення (хрестик) біля запису
4. Натисніть "Зберегти"
5. Якщо видалити всі записи -- процес стане доступним всім

## Приклади використання

| Сценарій | Налаштування ACL |
|----------|------------------|
| Процес для всіх | Немає записів (за замовчуванням) |
| Тільки HR-відділ | Додати відділ "HR" |
| Менеджери з будь-якого відділу | Додати роль "Менеджер" |
| HR або менеджери | Додати відділ "HR" + роль "Менеджер" |

## Корисні поради

- Використовуйте ACL для конфіденційних процесів (кадрові, фінансові)
- Комбінуйте відділи та ролі для гнучкого контролю
- Пам'ятайте: видалення всіх записів відкриває процес для всіх
- ACL перевіряється при спробі запуску та при завантаженні списку доступних процесів
