## Огляд

Система дозволів у Procivo побудована на моделі "ресурс + дія" з ієрархічним наслідуванням. Матриця дозволів дозволяє точно контролювати, хто і що може робити в організації, з підтримкою перевизначень на рівні конкретного користувача.

## Модель дозволів

Кожен дозвіл складається з двох компонентів:

- **Ресурс** -- тип об'єкта (задачі, процеси, дошки, визначення процесів тощо)
- **Дія** -- операція над ресурсом (перегляд, створення, редагування, видалення)

### Рівні доступу (scope)

Для кожної комбінації ресурс + дія можна встановити рівень:

| Рівень | Опис |
|--------|------|
| Немає | Доступ заборонений |
| Власні | Лише власні об'єкти |
| Відділ | Об'єкти свого відділу |
| Організація | Всі об'єкти організації |

## Ієрархія наслідування

Дозволи обчислюються за ієрархією (від загального до конкретного):

1. **Організація** -- базові дозволи для всіх
2. **Відділ** -- дозволи відділу, наслідують від батьківського відділу
3. **Роль** -- дозволи ролі працівника
4. **Користувач** -- індивідуальні перевизначення

Правила злиття:

- На одному рівні застосовується **ширший** доступ (наприклад, якщо дві ролі дають "відділ" і "організація", застосовується "організація")
- Перевизначення користувача **deny** блокує доступ негайно, незалежно від інших рівнів
- Перевизначення користувача **allow** розширює доступ понад рівень ролі

## Сторінка матриці дозволів

1. Перейдіть до розділу "Ролі та дозволи"
2. Оберіть роль або працівника
3. Відкриється таблиця: рядки -- ресурси, стовпці -- дії
4. У кожній комірці -- випадаючий список із рівнями доступу
5. Змініть потрібні рівні та збережіть

## Перевизначення на рівні користувача

1. Перейдіть до деталей працівника
2. Відкрийте вкладку "Дозволи"
3. Знайдіть потрібний ресурс + дію
4. Встановіть перевизначення:
   - **Allow** -- надати доступ, навіть якщо роль забороняє
   - **Deny** -- заборонити доступ, навіть якщо роль дозволяє
5. Зміни набувають чинності миттєво

## Корисні поради

- Налаштовуйте дозволи на рівні ролей, а не окремих користувачів -- це простіше підтримувати
- Використовуйте перевизначення лише у виняткових випадках
- Deny на рівні користувача -- найпотужніший інструмент обмеження
- Перевіряйте ефективні дозволи працівника після змін ієрархії
