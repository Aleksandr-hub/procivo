## Огляд

Кожна публікація визначення процесу створює нову версію -- незмінний знімок конфігурації. Запущені екземпляри прив'язані до версії, на якій вони стартували. Адміністратор може мігрувати екземпляри на нову версію з перевіркою сумісності.

## Створення версій

Версія створюється автоматично при публікації:

1. Відредагуйте визначення процесу в конструкторі
2. Натисніть "Опублікувати"
3. Система створює новий знімок (snapshot) з поточним станом вузлів та переходів
4. Версія отримує порядковий номер
5. Нові екземпляри будуть запускатися на цій версії

## Історія версій

1. Відкрийте визначення процесу
2. Натисніть іконку "Історія версій" (годинник)
3. Відкриється панель VersionHistoryDrawer зі списком всіх версій
4. Для кожної версії відображається:
   - Номер версії та дата створення
   - Кількість запущених екземплярів на цій версії
   - Стан версії (активна/архівна)

## Міграція екземплярів

Міграція переносить запущені екземпляри з однієї версії на іншу.

### Перевірка сумісності

Перед міграцією система перевіряє:

- Кожен активний вузол (де зараз знаходиться токен) повинен мати відповідний вузол у новій версії
- Активні задачі повинні мати вихідні переходи у новій версії
- Якщо перевірка не пройшла -- міграція блокується з поясненням

### Процес міграції

1. Відкрийте панель історії версій
2. Виберіть версію-джерело (з якої мігрувати)
3. Натисніть "Мігрувати на поточну версію"
4. Система виконає перевірку сумісності
5. При успіху -- екземпляри переносяться на нову версію
6. При помилці -- відображається список несумісних вузлів

### Event-Sourced міграція

Міграція реалізована через подійну модель:

- Для кожного екземпляра створюється подія `ProcessInstanceMigratedEvent`
- Подія записується в event store
- Проекція оновлює read model з новою версією
- Це забезпечує повну аудиторську простежуваність міграцій

## Корисні поради

- Завжди перевіряйте кількість активних екземплярів перед міграцією
- Тестуйте нову версію з новими екземплярами перед міграцією існуючих
- Якщо міграція блокується -- перевірте, чи активні задачі мають вихідні переходи
- Стара версія залишається доступною для перегляду в історії
- Міграція не можна скасувати -- плануйте уважно
