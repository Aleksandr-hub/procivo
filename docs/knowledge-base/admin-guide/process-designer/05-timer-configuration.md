## Огляд

Таймери дозволяють додавати затримки та дедлайни до процесів. Вузол таймера зупиняє виконання потоку на визначений час або до вказаної дати. Це корисно для автоматичних нагадувань, SLA та планування дій.

## Режими таймера

### Режим тривалості (Duration)

Таймер чекає визначений інтервал часу від моменту активації.

- **Формат:** ISO 8601 (наприклад, `PT2H` -- 2 години, `P3D` -- 3 дні)
- **Приклади:**
  | Значення | Тривалість |
  |----------|-----------|
  | `PT30M` | 30 хвилин |
  | `PT2H` | 2 години |
  | `PT1H30M` | 1 година 30 хвилин |
  | `P1D` | 1 день |
  | `P7D` | 7 днів |
  | `P1DT12H` | 1 день 12 годин |

### Режим дати (Date)

Таймер чекає до конкретної дати та часу.

- **Формат:** вибір дати та часу через DateTimePicker
- **Поведінка:** якщо дата вже минула, таймер спрацьовує негайно

## Налаштування в конструкторі

1. Перетягніть вузол "Таймер" на канвас
2. Виберіть вузол для відкриття панелі конфігурації
3. Оберіть режим: "Тривалість" або "Дата"
4. Для тривалості: задайте години, хвилини та дні через поля вводу
5. Для дати: оберіть дату та час через DateTimePicker
6. З'єднайте таймер переходами (один вхідний, один вихідний)

## Механізм виконання

Таймери виконуються через двоступеневий механізм:

1. **RabbitMQ** -- основний механізм: повідомлення з затримкою (delayed message) відправляється у чергу та обробляється воркером
2. **Консольна команда** -- резервний механізм: команда `FireOverdueTimersCommand` перевіряє таблицю `workflow_scheduled_timers` та запускає прострочені таймери

### Захист від подвійного спрацювання

- Таблиця `workflow_scheduled_timers` є джерелом правди
- Атомарний `UPDATE WHERE fired_at IS NULL` гарантує, що таймер спрацює лише один раз
- Навіть якщо RabbitMQ та консольна команда спрацюють одночасно -- конфлікту не буде

## Індикатори на запущених екземплярах

Запущені екземпляри процесу відображають стан таймерів:

- **Бейдж дедлайну** показується на вузлах зі статусом "очікує" (waiting)
- Колір індикатора змінюється при наближенні до дедлайну
- Завершені та скасовані токени не показують бейдж

## Корисні поради

- Використовуйте тривалість для відносних затримок (наприклад, "2 години після затвердження")
- Використовуйте дату для абсолютних дедлайнів (наприклад, "до кінця робочого дня")
- ISO 8601 формат: P (period) для днів, T (time) для годин/хвилин
- Перевіряйте таймери на коротких інтервалах (PT1M) перед використанням великих значень
