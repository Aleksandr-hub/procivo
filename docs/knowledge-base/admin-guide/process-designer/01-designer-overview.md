## Огляд

Конструктор процесів -- це візуальний інструмент для проектування BPMN-процесів у Procivo. Він дозволяє створювати та редагувати визначення процесів: додавати вузли (задачі, шлюзи, таймери), з'єднувати їх переходами та налаштовувати поведінку кожного елемента.

## Доступ до конструктора

1. Перейдіть до розділу "Процеси" в головному меню
2. Відкрийте список визначень процесів
3. Натисніть "Створити визначення" для нового процесу, або натисніть "Редагувати" на існуючому
4. Відкриється канвас конструктора

## Інтерфейс конструктора

### Канвас

Основна робоча область, де розміщуються вузли та переходи:

- **Перетягування** вузлів для зміни позиції
- **З'єднання** вузлів переходами (клік на вихідний порт та перетягування до цільового вузла)
- **Масштабування** колесом миші
- **Переміщення** канвасу перетягуванням фону

### Панель інструментів

Зліва розташована панель з типами вузлів:

- Старт (Start)
- Кінець (End)
- Задача (Task)
- Шлюз XOR (XOR Gateway)
- Таймер (Timer)

Перетягніть потрібний тип на канвас для додавання.

### Панель конфігурації

При виборі вузла або переходу справа з'являється панель налаштувань:

- Назва та опис вузла
- Тип призначення (для задач)
- Форми дій (для переходів)
- Умови маршрутизації (для шлюзів)

## Збереження та публікація

### Збереження

- Натисніть "Зберегти" для збереження поточного стану
- Кожне збереження зберігає чернетку визначення
- Зміни не впливають на запущені процеси

### Публікація (Deploy)

- Натисніть "Опублікувати" для створення нової версії
- Публікація створює незмінний знімок (snapshot) визначення
- Нові екземпляри процесу запускаються на останній опублікованій версії
- Кнопка "Опублікувати" завжди доступна -- натискання на вже опублікованому визначенні створює нову версію

### Стани визначення процесу

| Стан | Опис |
|------|------|
| Чернетка (Draft) | Визначення в процесі редагування, не можна запускати |
| Опубліковано (Published) | Визначення опубліковано, можна запускати нові екземпляри |

## Корисні поради

- Почніть з простого процесу: Старт -> Задача -> Кінець
- Перевіряйте валідність перед публікацією -- канвас підсвітить помилки
- Публікація не зупиняє запущені екземпляри -- вони продовжують на своїй версії
- Використовуйте описові назви вузлів для зрозумілості процесу
