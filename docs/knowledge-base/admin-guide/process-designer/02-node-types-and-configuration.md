## Огляд

Процес складається з вузлів, з'єднаних переходами. Кожен тип вузла має своє призначення та набір налаштувань. Правильне використання вузлів дозволяє моделювати складні бізнес-процеси.

## Start (Стартовий вузол)

Точка входу в процес. Кожне визначення має рівно один стартовий вузол.

- **Конфігурація:** назва (необов'язково)
- **Переходи:** лише вихідні (один або більше)
- **Поведінка:** при запуску екземпляра процесу токен створюється у стартовому вузлі та переміщується далі

## End (Кінцевий вузол)

Точка завершення процесу. Визначення може мати кілька кінцевих вузлів.

- **Конфігурація:** назва (необов'язково)
- **Переходи:** лише вхідні
- **Поведінка:** коли токен досягає кінцевого вузла, екземпляр процесу завершується

## Task (Вузол задачі)

Користувацька задача, яка вимагає дії від призначеного виконавця.

- **Конфігурація:**
  - Назва задачі (обов'язково)
  - Опис (необов'язково)
  - Стратегія призначення (хто виконує)
  - Форми дій (на кожному вихідному переході)
- **Переходи:** вхідні та вихідні
- **Поведінка:** токен зупиняється у вузлі задачі та чекає, поки виконавець обере дію (перехід)

### Приклад

Задача "Перевірити заявку" з двома вихідними переходами:
- "Затвердити" -- перехід до наступного кроку
- "Відхилити" -- перехід до завершення

## XOR Gateway (Шлюз XOR)

Умовний шлюз для маршрутизації потоку на основі виразів.

- **Конфігурація:**
  - Назва шлюзу (необов'язково)
  - Умовні вирази на кожному вихідному переході (Symfony ExpressionLanguage)
- **Переходи:** один вхідний, два або більше вихідних
- **Поведінка:** токен перевіряє умови переходів зверху вниз, і переміщується через перший перехід, де умова істинна

### Приклад виразів

| Перехід | Умова |
|---------|-------|
| Великий бюджет | `context['budget'] > 10000` |
| Малий бюджет | `context['budget'] <= 10000` |

## Timer (Таймер)

Вузол затримки, який зупиняє токен на визначений час.

- **Конфігурація:**
  - Режим: тривалість (duration) або дата (date)
  - Значення: ISO 8601 інтервал або конкретна дата
- **Переходи:** один вхідний, один вихідний
- **Поведінка:** токен чекає до закінчення таймера, потім автоматично переміщується далі

## З'єднання вузлів переходами

1. Наведіть курсор на вихідний порт вузла (зелена точка)
2. Натисніть та перетягніть до вхідного порту цільового вузла
3. Відпустіть -- перехід створено
4. Натисніть на перехід для налаштування форми або умови

### Правила з'єднання

- Стартовий вузол: лише вихідні переходи
- Кінцевий вузол: лише вхідні переходи
- Задача: вхідні та вихідні
- XOR шлюз: один вхідний, мінімум два вихідних
- Таймер: один вхідний, один вихідний

## Корисні поради

- Давайте зрозумілі назви вузлам -- вони відображаються у списку задач виконавців
- XOR шлюз оцінює умови зверху вниз -- ставте найімовірнішу умову першою
- Кожна задача повинна мати хоча б один вихідний перехід
- Використовуйте таймер для автоматичних дедлайнів та затримок
