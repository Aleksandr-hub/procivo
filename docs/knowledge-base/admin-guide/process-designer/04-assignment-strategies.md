## Огляд

Стратегія призначення визначає, хто виконуватиме задачу у процесі. Procivo підтримує 4 стратегії, від прямого призначення конкретній особі до динамічного призначення з контексту процесу.

## Стратегії

### 1. Конкретний користувач (Specific User)

Задача призначається обраному працівнику.

- **Налаштування:** виберіть працівника зі списку організації
- **Поведінка:** задача одразу з'являється у списку задач обраного працівника
- **Коли використовувати:** для задач, які завжди виконує одна конкретна особа

### 2. Пул ролі (Role Pool)

Задача доступна всім працівникам з визначеною роллю.

- **Налаштування:** виберіть роль зі списку ролей організації
- **Поведінка:** задача відображається як "пулова" для всіх з цією роллю
- **Виконавець:** будь-хто з ролі може "претендувати" (claim) на задачу
- **Коли використовувати:** для задач, які може виконати будь-хто з певної групи

### 3. Пул відділу (Department Pool)

Задача доступна всім працівникам визначеного відділу.

- **Налаштування:** виберіть відділ зі списку відділів організації
- **Поведінка:** аналогічно пулу ролі, але фільтрація за відділом
- **Коли використовувати:** для задач, специфічних для конкретного підрозділу

### 4. Зі змінної (From Variable)

Виконавець визначається зі змінної контексту процесу під час виконання.

- **Налаштування:** вкажіть назву змінної контексту (наприклад, `assigneeId`)
- **Поведінка:** при активації задачі система зчитує значення змінної та призначає задачу вказаному користувачу
- **Коли використовувати:** коли виконавець визначається на попередніх кроках процесу

## Пулові задачі (Pool Tasks)

Коли задача призначена пулу (ролі або відділу), вона з'являється як "пулова":

### Банер пулової задачі

- Задача відображає банер PoolTaskBanner із повідомленням "Ця задача доступна для претендування"
- Кожен учасник пулу бачить задачу у своєму списку

### Претендування (Claim)

1. Знайдіть пулову задачу у списку задач
2. Відкрийте деталі задачі
3. Натисніть "Претендувати" (Claim)
4. Задача стає призначеною вам -- інші учасники пулу більше її не бачать

### Відмова від претендування (Unclaim)

1. Відкрийте задачу, на яку ви претендуєте
2. Натисніть "Відпустити" (Unclaim)
3. Задача повертається у пул та стає доступною іншим

### Песимістичне блокування

Система використовує песимістичне блокування для запобігання конфліктам:

- Якщо два працівники одночасно натискають "Претендувати", лише один отримає задачу
- Другий побачить повідомлення, що задача вже призначена

## Налаштування в конструкторі

1. Виберіть вузол задачі на канвасі
2. У панелі конфігурації знайдіть "Стратегія призначення"
3. Оберіть одну з 4 стратегій
4. Заповніть додаткові параметри залежно від стратегії
5. Збережіть визначення

## Корисні поради

- Використовуйте пул ролі для задач, які можуть виконати кілька працівників
- Пул відділу зручний для задач, специфічних для підрозділу
- From Variable корисний для ланцюжків задач, де виконавець визначається динамічно
- Пулові задачі підвищують ефективність -- перша вільна людина забирає задачу
