## Огляд

Форми в Procivo прив'язані до дій (переходів), а не до задач. Кожен перехід з вузла задачі може мати власну форму з набором полів. Це дозволяє збирати різні дані залежно від обраної дії: наприклад, "Затвердити" потребує лише коментар, а "Відхилити" -- причину відхилення.

## Принцип "форма на дію"

- Кожен **перехід** (transition) з вузла задачі може мати свою форму
- Якщо перехід не має форми -- виконавець просто натискає кнопку дії
- Якщо перехід має форму -- відкривається діалог із полями для заповнення
- Дані форми зберігаються в контексті процесу та доступні в подальших кроках

## Додавання полів до переходу

1. В конструкторі натисніть на перехід (стрілку між вузлами)
2. У панелі конфігурації справа знайдіть секцію "Поля форми"
3. Натисніть "Додати поле"
4. Оберіть тип поля
5. Налаштуйте назву, підпис та обмеження
6. Повторіть для додаткових полів

## Типи полів

| Тип | Опис | Приклад |
|-----|------|---------|
| Text | Однорядковий текст | Ім'я, номер документа |
| Number | Числове значення | Бюджет, кількість |
| Date | Вибір дати | Дедлайн, дата народження |
| Select | Випадаючий список | Статус, категорія |
| Checkbox | Прапорець (так/ні) | Підтвердження, згода |
| Textarea | Багаторядковий текст | Коментар, опис |

## Обмеження (Constraints)

Для кожного поля можна задати обмеження валідації:

| Обмеження | Опис | Приклад |
|-----------|------|---------|
| Required | Обов'язкове поле | Будь-яке поле |
| Min | Мінімальне значення / довжина | Число >= 0 |
| Max | Максимальне значення / довжина | Текст <= 500 символів |
| Regex | Регулярний вираз | Email, номер телефону |

### Налаштування обмежень

1. Виберіть поле у списку
2. Знайдіть секцію "Обмеження"
3. Увімкніть потрібне обмеження перемикачем
4. Задайте значення (для min/max/regex)
5. Обмеження перевіряються при заповненні форми виконавцем

## Залежності полів

Поля можуть залежати одне від одного:

- Поле B відображається, лише якщо поле A має певне значення
- Наприклад: поле "Причина відхилення" з'являється лише якщо "Рішення" = "Відхилити"

Налаштування залежностей:

1. Оберіть поле, яке має бути залежним
2. У секції "Залежності" натисніть "Додати залежність"
3. Оберіть поле-батько та значення, при якому залежне поле відображається
4. Збережіть

## Попередній перегляд форми

Натисніть "Попередній перегляд" у панелі конфігурації переходу, щоб побачити, як форма виглядатиме для виконавця.

## Корисні поради

- Мінімізуйте кількість полів -- запитуйте лише необхідні дані
- Використовуйте описові підписи для кожного поля
- Обов'язкові поля позначайте зірочкою (автоматично при required)
- Дані форми доступні через `context['fieldName']` у виразах XOR-шлюзів
- Для складних форм використовуйте залежності замість відображення всіх полів
